## Analiza: AKO vs Single

### Matematyka
```
EV_AKO = (1+EV₁)(1+EV₂)... - 1 = EV₁ + EV₂ + EV₁×EV₂
```
Człon `EV₁×EV₂ > 0` gdy oba EV pozytywne → **AKO amplikuje edge**.

Przykład (2 nogi EV=+33%, +30%, 10 PLN):
- 2x single: EV = +3.15 PLN
- 1x AKO:   EV = +7.29 PLN (2.3× wyższy!)

### Kiedy AKO > Single
- Model **skalibrowany** (hit rate > 45% przy EV > 0)
- Bankroll pozwala przeżyć wariancję

### Kiedy Single > AKO (obecny stan)
- Hit rate = 0% → model nieskalib.
- AKO amplikuje błędy modelu tak samo jak edge
- Single limitują straty podczas nauki

### Wniosek
Program słusznie używa single teraz. Po zebraniu 20-30 zakończonych typów
i weryfikacji hit rate → przejście na AKO z pozytywnym EV per noga = lepszy EV.
